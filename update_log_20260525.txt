# 2026-05-25 업데이트 내역

1. **디스크 백업 복구 (폴더 백업 불러오기) 이미지 404 에러 해결**
   - 문제 원인: 자동 저장(`workspace_autosave.json`) 시 용량 문제로 이미지 데이터가 `[autosaved-image]`로 치환되어 디스크에 저장됨. 이 파일을 그대로 복구하면 이미지가 유실됨.
   - 해결 방법: 니체 1권, 탈무드 1권에 대해 로컬 폴더(pages)에 있는 실제 이미지(png, jpg) 경로를 탐색하여 강제로 매핑해주는 Node 스크립트를 작성하여 `workspace_autosave_fixed.json` 생성.
   - `recovery.jsx`에서 `_fixed.json`이 존재하면 우선적으로 불러와 이미지가 깨지지 않고 복구되도록 수정.

2. **PDF 및 이미지 일괄 생성 버튼 통합**
   - 문제: '책구조'와 '미리보기' 탭에 PDF/이미지 생성 버튼이 4개씩 분산되어 있어 일일이 클릭해야 하는 불편함.
   - 해결 방법:
     - '책구조' 탭의 인쇄용 PDF(한/영) 및 5스프레드 작업본문 이미지(한/영) 생성 버튼 4개를 하나의 **[인쇄용 PDF / 작업본문 이미지 (4개 파일 일괄 생성)]** 버튼으로 통합 (`exportAllStructure` 함수 추가).
     - '미리보기' 탭의 16p PDF(한/영) 및 카드 PDF(한/영) 생성 버튼 4개를 하나의 **[모든 미리보기 PDF (4개 파일 일괄 생성)]** 버튼으로 통합.
   - 알림(alert) 최소화: 각 파일별로 생성 성공 알림이 뜨던 것을 일괄 생성 모드(`opts.silent`)에서는 생략하고, 4개 작업이 모두 끝난 뒤 1번의 완료 알림만 뜨도록 수정.
   - 렌더링 에러(`exportAllStructure is not defined`) 해결: `BookGrid` 스코프 내에 올바르게 함수를 배치하여 문제 해결.
